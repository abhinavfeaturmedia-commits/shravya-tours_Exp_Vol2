# Directives

SOPs (Standard Operating Procedures) written in Markdown. Each directive defines:

- **Goal** — What needs to be accomplished
- **Inputs** — What data or context is needed
- **Tools / Scripts** — Which `execution/` scripts to use
- **Outputs** — Expected deliverables
- **Edge Cases** — Known pitfalls and how to handle them

Directives are living documents — they get updated whenever we learn something new (API limits, better approaches, gotchas).
